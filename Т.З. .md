## 2. 🎯 Цель и Задачи

**Цель:** Создать автономное веб-приложение для проведения игры «Поле чудес» без необходимости настройки сервера, обеспечивающее строгий тайминг и управление игровыми механиками.

**Задачи:**

1. Реализовать игровую логику (барабан, буквы, очки).
2. Обеспечить сохранение прогресса между играми (5 отборочных + финал).
3. Создать интерфейс для ведущего (управление) и зрителей (табло).
4. Гарантировать работу без интернета (опционально, PWA).

## 5. 🧩 Игровая механика (Business Logic)

### 5.1. Сектора барабана (Вероятности)

Реализовать взвешенную случайность (Weighted Random):


| Сектор    | Вес (%) | Действие                                           |
| --------------- | ---------- | ---------------------------------------------------------- |
| 1-5 очков  | 40%        | Начисление очков                            |
| 6-10 очков | 20%        | Начисление очков                            |
| x2              | 10%        | Удвоение текущего хода                 |
| +               | 10%        | Дополнительный ход                        |
| БАНКРОТ  | 10%        | Потеря хода + сброс очков раунда |
| ПРИЗ        | 5%         | Виртуальный приз (анимация)         |
| БАНК        | 5%         | Сохранение очков                            |

### 5.2. Тайминг

* **Ход игрока:** 15 секунд жесткого таймера.
* **Визуализация:** Прогресс-бар вокруг кнопки или отдельный таймер.
* **Звук:** Тиканье (опционально), сигнал за 3 сек, сигнал 0 сек.
* **Действие по 0:** Автоматический переход хода (или кнопка «Пропуск» у ведущего).

### 5.3. Система очков

* Очки суммируются внутри одной игры.
* Между играми очки сбрасываются (кроме финала, если решим иначе, но по плану — каждая игра независима для выхода в финал).
* **Победа в раунде:** Тот, кто отгадал слово ИЛИ у кого больше очков, если слово не отгадано (но лучше стимулировать отгадывание).
* **Рекомендация:** Победа только при отгадывании слова. Если слово не отгадано — ведущий открывает его вручную и выбирает победителя по очкам.

3. Кнопка «Загрузить бэкап» (восстанавливает состояние из JSON).

## 7. 🎨 UI/UX Требования

### 7.1. Дизайн-система

* **Стиль:** Корпоративный, праздничный (темно-синий/золотой или цвета бренда компании).
* **Шрифты:** Крупные, без засечек (Roboto, Montserrat), читаемые с 5 метров.
* **Разрешение:** Адаптив под 1920x1080 (Full HD).
* **Цвета:**
  * Фон: Темный (для экономии энергии проектора и контраста).
  * Буквы: Яркие (желтый/белый).
  * Акценты: Зеленый (верно), Красный (ошибка/банкрот).

### 7.2. Экраны

1. **Dashboard (Ведущий):**
   * Видно только на ноутбуке (или скрытая панель).
   * Кнопки управления барабаном, буквами, таймером.
   * Ввод названных букв.
2. **Stage (Проектор):**
   * Только табло, вопрос, счет, анимация барабана.
   * Никаких кнопок управления.
   * *Техническое решение:* Режим «Второй экран» или просто скрытие панели управления через CSS класс `.admin-panel { display: none }`.

### 7.3. Анимации

* Вращение барабана (CSS animation / JS Canvas).
* Появление буквы (Fade-in + Scale).
* Таймер (Circular progress bar).

## 9. ✅ Критерии приемки (Acceptance Criteria)

1. **Запуск:** Приложение открывается в браузере без ошибок консоли.
2. **Настройка:** Можно ввести 5 групп и 6 слов за ≤ 3 минут.
3. **Игра:**
   * Барабан крутится, сектор выбирается случайно.
   * Таймер отсчитывает 15 секунд и издает звук.
   * Буквы открываются корректно во всех позициях.
   * Счет обновляется правильно.
4. **Сохранение:**
   * При обновлении страницы (F5) прогресс не теряется.
   * Функция Export/Import JSON работает корректно.
5. **Финал:** После 5-й игры доступен переход к 6-й (Финал).
6. **Производительность:** FPS ≥ 60 при анимации барабана, задержка ввода < 100мс.

## 11. 📅 Этапы разработки (Roadmap)

1. **Прототип (2 дня):** Верстка табло, логика открытия букв.
2. **Логика (3 дня):** Барабан, очки, таймер, переходы ходов.
3. **Данные (1 день):** LocalStorage, импорт/экспорт.
4. **UI/UX (2 дня):** Анимации, звуки, адаптация под проектор.
5. **Тестирование (1 день):** Прогон сценария 6 игр, проверка таймингов.
6. **Финализация (1 день):** Исправление багов, сборка.

**Итого:** ~10 рабочих дней (2 недели).

## 12. 📝 Примечания для разработчика

1. **Offline First:** Приложение должно работать без подключения к интернету (все ассеты локально).
2. **Звук:** Предусмотреть кнопку Mute (на корпоративе может быть свой звук).
3. **Клавиатура:** Добавить горячие клавиши для ведущего (например, `Space` — крутить барабан, `Enter` — подтвердить букву).
4. **Безопасность:** Так как все на фронтенде, пользователь может изменить очки в консоли. Это допустимо для развлекательного приложения, но можно добавить простую_obfuscation_ для состояния.
